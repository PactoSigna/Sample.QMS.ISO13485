# WI-001: GitHub Pull Request Review

## Purpose

Step-by-step instructions for reviewing document and code changes via GitHub Pull Requests.

## Prerequisites

- GitHub account with repository access
- Understanding of document being reviewed

## Procedure

### For Reviewers

1. **Open the Pull Request**
   - Navigate to the repository on GitHub
   - Click "Pull requests" tab
   - Select the PR to review

2. **Review Changes**
   - Click "Files changed" tab
   - Review each modified file
   - Check for:
     - Technical accuracy
     - Completeness
     - Proper formatting
     - Valid traceability links

3. **Provide Feedback**
   - Click "+" next to line to add comment
   - Use "Start a review" to batch comments
   - Be specific and constructive

4. **Submit Review**
   - Click "Review changes"
   - Select: Approve, Request changes, or Comment
   - Submit review

### For Authors

1. **Address Feedback**
   - Review all comments
   - Make requested changes
   - Reply to comments explaining resolution

2. **Request Re-review**
   - Push updated commits
   - Request re-review from reviewers
